## 1. 应用程序概要

### 1.1 基本信息

| 属性 | 内容 |
|------|------|
| **应用名称** | 星屿 StarIsle |
| **应用版本** | MVP v1.3 |
| **应用类型** | 青少年心理健康AI陪伴应用（多端） |
| **目标用户** | 12-18岁初高中生、教师、家长 |
| **核心定位** | AI情绪成长伙伴，零压力的第一心理求助站 |
| **品牌Slogan** | 「你的情绪星球，永远亮着灯」 |

### 1.2 应用功能概述

星屿 StarIsle 是专为12-18岁初高中生打造的AI心理健康应用，通过极简心情打卡和24/7 AI对话，为学生提供零压力的情绪支持。同时为教师提供心理守护协同工作台，为家长提供孩子情绪状态查看与AI心理咨询服务，实现家校共育。

### 1.3 多端架构

| 端 | 技术栈 | 部署形态 | 核心功能 |
|------|--------|---------|---------|
| **学生端** | Flutter 3.x (Dart) | iOS / Android 原生APP | 心情打卡、AI星宝对话、情绪测评、冥想放松、风险检测 |
| **教师端** | Flutter 3.x (Dart) | iOS / Android 原生APP | 工作台告警、学生列表、对话观察、症状反馈 |
| **家长端** | React 18 + TypeScript | Web应用（Capacitor跨平台） | 情绪查看、大星AI对话、应急预案、知识库 |
| **Web前端** | React 18 + Vite + TailwindCSS | Web应用 | 统一入口，支持三端角色切换 |
| **后端服务** | Java (Spring Boot 3.2) | 服务器部署 | RESTful API、WebSocket、JWT认证、RBAC授权 |
| **AI引擎** | Python (FastAPI) | 服务器部署 | 对话生成、情绪分析、风险检测、语义分析 |

## 6. MITRE ATT&CK 评估范围声明

### 6.1 评估目标

本次安全评估基于 MITRE ATT&CK 框架，对星屿 StarIsle 应用程序进行全面安全态势评估，包括：

1. **威胁建模**: 基于 ATT&CK 矩阵识别潜在攻击向量
2. **安全控制映射**: 将现有安全控制措施映射到 ATT&CK 技术
3. **覆盖率分析**: 评估安全控制对 ATT&CK 技术的覆盖程度
4. **差距识别**: 识别安全控制覆盖不足的领域
5. **改进建议**: 提出基于 ATT&CK 框架的安全改进建议

### 6.2 适用矩阵

| ATT&CK矩阵 | 适用性 | 说明 |
|------------|--------|------|
| **Enterprise** | 适用 | 后端服务器、API网关、数据库 |
| **Cloud** | 适用 | 云部署环境、容器化服务 |
| **Mobile** | 适用 | 学生端/教师端 Flutter 移动应用 |
| **ICS** | 不适用 | 非工业控制系统 |

### 6.3 评估说明

> **重要说明**: MITRE ATT&CK Evaluations (ER8) 官方注册已于 2026年4月30日 截止，且该评估主要面向安全产品厂商（EDR/XDR/SIEM等）。星屿 StarIsle 作为应用程序，本次评估采用 MITRE ATT&CK 框架作为威胁建模和安全评估的方法论工具，对应用自身的安全态势进行系统性评估，而非参加 MITRE 官方评估认证。评估结果将作为内部安全改进的依据，并可作为第三方安全审计的参考材料。
