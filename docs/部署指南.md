## 部署步骤

### 1. 环境准备

#### 系统要求
- **服务器**: 4核8GB内存（最低配置）
- **操作系统**: Ubuntu 20.04 LTS
- **Docker**: 20.10+
- **Kubernetes**: 1.25+（可选）

#### 软件依赖
- Docker & Docker Compose
- Kubernetes（生产环境）
- Nginx（负载均衡）
- SSL证书

### 2. 本地开发环境部署

#### 步骤1: 克隆项目
```bash
git clone https://github.com/user-unknowed/-StarIsle-.git
cd -StarIsle-
```

#### 步骤2: 配置环境变量
```bash
cp .env.template .env
# 编辑.env文件，填入真实配置值
nano .env
```

#### 步骤3: 启动Docker容器
```bash
cd deployment
docker-compose up -d
```

#### 步骤4: 验证服务状态
```bash
docker-compose ps
curl http://localhost:8080/health
```

### 3. 生产环境部署

#### 选项A: Kubernetes部署（推荐）

#### 步骤1: 创建命名空间
```bash
kubectl create namespace starisle
```

#### 步骤2: 部署Secret配置
```bash
kubectl apply -f deployment/kubernetes/starisle-deployment.yml
```

#### 步骤3: 验证部署状态
```bash
kubectl get pods -n starisle
kubectl get services -n starisle
```

#### 选项B: Docker Swarm部署

```bash
docker swarm init
docker stack deploy -c deployment/docker-compose.yml starisle
```

### 4. 数据库初始化

#### PostgreSQL初始化
```bash
docker exec -i starisle-postgres psql -U starisle_user -d starisle < database/postgres/init.sql
```

#### MongoDB初始化
```bash
# MongoDB会自动通过环境变量初始化
```

### 5. 前端应用部署

#### Flutter Web构建
```bash
cd frontend
flutter build web --release
```

#### 部署到CDN/Nginx
```bash
# 将构建产物复制到Nginx静态资源目录
cp -r build/web/* /var/www/starisle/
```

### 6. SSL证书配置

#### 使用Let's Encrypt
```bash
sudo certbot --nginx -d starisle.example.com
```

#### 手动配置证书
```bash
# 将证书放置到Nginx证书目录
sudo cp starisle.crt /etc/nginx/ssl/
sudo cp starisle.key /etc/nginx/ssl/
```

### 7. 监控与日志

#### Prometheus监控
```bash
kubectl apply -f deployment/monitoring/prometheus.yml
```

#### Grafana可视化
```bash
kubectl apply -f deployment/monitoring/grafana.yml
```

#### ELK日志收集
```bash
kubectl apply -f deployment/logging/elk.yml
```

## 常见问题

### Q1: 如何备份数据库？
```bash
# PostgreSQL备份
pg_dump -U starisle_user starisle > backup_$(date +%Y%m%d).sql

# MongoDB备份
mongodump --uri="mongodb://starisle_admin:password@localhost:27017/starisle"
```

### Q2: 如何更新应用？
```bash
# Kubernetes滚动更新
kubectl set image deployment/api-gateway api-gateway=starisle/api-gateway:v1.1

# Docker Swarm更新
docker service update --image starisle/api-gateway:v1.1 starisle_api-gateway
```

### Q3: 如何扩展服务？
```bash
# Kubernetes扩展副本
kubectl scale deployment api-gateway --replicas=5

# Docker Swarm扩展
docker service scale starisle_api-gateway=5
```

## 安全配置

### 1. 网络安全
- 配置防火墙规则，只开放必要端口（80, 443, 8080）
- 使用TLS 1.3加密传输
- 配置IP白名单

### 2. 数据安全
- 定期数据库备份
- 启用数据库加密存储
- 配置数据库访问控制

### 3. 应用安全
- 定期更新依赖包
- 启用安全headers
- 配置速率限制

## 性能优化

### 1. 数据库优化
- 配置数据库连接池
- 启用查询优化
- 定期清理过期数据

### 2. 缓存优化
- Redis缓存热点数据
- CDN缓存静态资源
- 启用HTTP缓存headers

### 3. 应用优化
- 启用Gzip压缩
- 优化图片资源
- 启用懒加载
