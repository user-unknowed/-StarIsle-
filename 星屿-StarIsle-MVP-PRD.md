## 1. 文档概述

### 1.1 文档目的

本文档为「星屿」StarIsle MVP 阶段（第1-4个月）的产品需求文档，用于指导设计、研发和测试团队完成 MVP 版本的交付。

### 1.2 MVP 范围定义

| 包含 | 不包含 |
|------|--------|
| AI 星宝基础对话 | 咨询师匹配与转介 |
| 心情打卡 | 家长端 & 学校端 |
| 情绪探索（1个基础测评） | 匿名树洞社区 |
| 基础冥想（3-5个音频） | CBT 自助练习库 |
| 风险检测与危机响应 | 游戏化成长体系 |
| 学生端 iOS + Android APP | 微信小程序 |
| 隐私基础（匿名注册、加密） | 情绪报告（周报/月报） |

### 1.3 术语约定

| 术语 | 说明 |
|------|------|
| 星宝 | APP 内置的 AI 情绪伙伴，基于大模型的对话机器人 |
| 心情打卡 | 用户每日记录心情的极简操作，5 档表情选择 |
| 情绪探索 | 去标签化的心理健康评估，底层映射 PHQ-9 量表 |
| 星屿 | 产品名称，也是用户个人空间的隐喻 |
| 风险等级 | 绿/黄/橙/红四级情绪风险评估 |

## 2. 产品背景与目标

### 2.1 背景

- 中国 14-18 岁青少年抑郁检出率约 **24.6%**，高中生接近 **40%**，但专业心理服务使用率仅 **7.4%**
- 求助最大障碍为**病耻感**和**隐私恐惧**，超过 30% 有困扰的学生不愿主动求助
- 学校心理咨询室使用率低，学生担心被标签化和隐私泄露
- 教育部 2025 年《十条措施》明确要求利用 AI 技术赋能学生心理健康工作
- 国内专门面向 12-18 岁初高中生的 AI 心理健康 APP 几乎是空白市场

### 2.2 MVP 目标

**核心假设**：青少年是否愿意使用 AI 情绪伙伴来获得心理支持？

| 指标 | 目标值 |
|------|--------|
| 7 日留存率 | > 30% |
| 30 日留存率 | > 15% |
| 日均心情打卡率（打卡用户/DAU） | > 40% |
| 用户 NPS | > 40 |
| 高风险用户危机热线触达率 | > 80% |

### 2.3 产品定位（MVP）

> **为 12-18 岁初高中生打造的 AI 情绪陪伴工具，通过极简的心情打卡和 24/7 AI 对话，让学生在零压力下获得第一时间的情绪支持。**

## 4. 核心功能清单（EARS 原则）

### 4.1 模块一：注册与隐私基础

#### FR-1.1 匿名注册

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Event-driven |
| **需求描述** | When 用户首次打开 APP，the system shall 展示品牌动画后进入隐私说明页，以漫画形式说明数据保护措施，然后引导用户创建匿名身份（昵称 + 头像 + 年龄段），无需手机号或实名认证 |
| **验收标准** | 1. 完成注册流程不超过 3 分钟；2. 注册仅需昵称、头像、年龄段三项信息；3. 提供不少于 30 个头像选项 |
| **优先级** | P0 |

#### FR-1.2 隐私说明

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Ubiquitous |
| **需求描述** | The system shall 在注册流程中以漫画/动画形式向用户说明：①日记仅自己可见；②与星宝的对话端到端加密；③老师和家长无法查看内容（除非用户主动授权）；④高风险时系统会触发预警保护机制 |
| **验收标准** | 1. 隐私说明页阅读完成率 > 90%；2. 隐私说明不超过 5 屏；3. 语言符合青少年阅读水平 |
| **优先级** | P0 |

#### FR-1.3 隐私设置

| 属性 | 内容 |
|------|------|
| **EARS 类型** | State-driven |
| **需求描述** | While 用户已登录，the system shall 在"我的"页面提供隐私设置入口，允许用户查看数据收集清单、导出个人数据、一键删除账号及全部数据 |
| **验收标准** | 1. 数据删除请求在 7 个工作日内完成；2. 数据导出格式为可读的 JSON/PDF |
| **优先级** | P0 |

### 4.2 模块二：心情打卡

#### FR-2.1 极简心情打卡

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Event-driven |
| **需求描述** | When 用户点击首页打卡按钮，the system shall 展示 5 档心情表情（😄 很棒 / 🙂 不错 / 😐 一般 / 😔 不太好 / 😢 很糟），用户 1 次点击即可完成打卡。打卡后可选择补充标签（学习压力/人际/家庭/孤独/其他），标签为非必选。整个流程不超过 10 秒 |
| **验收标准** | 1. 从点击到完成打卡 ≤ 3 次点击（含表情选择）；2. 打卡完成有明确视觉反馈（动画 + 音效）；3. 标签选择为非阻断式（可跳过） |
| **优先级** | P0 |

#### FR-2.2 打卡连续性激励

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Event-driven |
| **需求描述** | When 用户完成每日打卡，the system shall 展示连续打卡天数，并在首页显示打卡进度条。连续打卡 7 天、14 天、30 天时展示成就提示 |
| **验收标准** | 1. 打卡进度条在首页始终可见；2. 成就提示为非阻断式（轻提示/Toast） |
| **优先级** | P0 |

#### FR-2.3 智能追问

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Event-driven |
| **需求描述** | When 用户连续 3 天选择"不太好"或"很糟"，the system shall 通过星宝发起温和的关怀式对话，引导用户进行更深度的情绪探索或推荐放松内容 |
| **验收标准** | 1. 触发准确率 100%；2. 关怀话术温和、非侵入式，不引起用户反感 |
| **优先级** | P1 |

#### FR-2.4 心情晴雨表

| 属性 | 内容 |
|------|------|
| **EARS 类型** | State-driven |
| **需求描述** | While 用户在首页，the system shall 展示最近 7 天的心情趋势图（柱状图/折线图），让用户直观看到自己的情绪变化 |
| **验收标准** | 1. 趋势图在首页首屏可见（不需要滚动）；2. 支持周视图切换；3. 无数据时不显示空白图表 |
| **优先级** | P0 |

### 4.3 模块三：AI 星宝对话

#### FR-3.1 基础对话能力

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Event-driven |
| **需求描述** | When 用户进入聊天页与星宝对话，the system shall 基于大语言模型提供共情优先的对话回应：①始终先回应情绪再提供建议；②基于 CBT 框架帮助用户识别自动负性思维；③引导积极的替代性思维；④对话风格温暖、非评判、贴近青少年表达 |
| **验收标准** | 1. 对话响应延迟 < 3 秒（P95）；2. 共情语句出现在每次回应的开头；3. 不出现诊断性/标签化语言（如"抑郁""焦虑症"等）；4. 通过 50 个标准测试用例的对话质量评审 |
| **优先级** | P0 |

#### FR-3.2 话题引导卡片

| 属性 | 内容 |
|------|------|
| **EARS 类型** | State-driven |
| **需求描述** | While 用户在对话界面且超过 10 秒未输入，或用户主动点击"不知道聊什么"，the system shall 展示话题引导卡片，提供 3-5 个可选话题（如"聊聊最近的压力""关于朋友的事""未来让我焦虑""和家人相处""没有什么特别的事，就是有点闷"）。用户点击卡片后，星宝以该话题开启对话 |
| **验收标准** | 1. 话题卡片在对话界面底部区域呈现，不遮挡对话历史；2. 每次展示的话题至少 3 个；3. 话题卡片可关闭 |
| **优先级** | P1 |

#### FR-3.3 沉默关怀

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Event-driven |
| **需求描述** | When 用户超过 3 天未与星宝对话且历史心情记录偏负面（近 7 天有 ≥ 3 天选择"不太好/很糟"），the system shall 通过系统通知（APP 推送）发送关怀消息，内容为温和的邀请式语言，如"星宝有点想你了，最近过得怎么样？" |
| **验收标准** | 1. 触发条件准确率 100%；2. 推送通知可被用户在设置中关闭；3. 推送频率不超过每 3 天 1 次 |
| **优先级** | P1 |

#### FR-3.4 对话安全边界

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Ubiquitous |
| **需求描述** | The system shall 在每次星宝对话中遵循以下安全规则：①绝不提供自伤/自杀方法的任何信息；②在对话界面始终显示"紧急帮助"入口按钮；③当检测到高风险关键词时自动触发危机干预流程（见 FR-5.1）；④首次对话中明确告知"星宝是 AI 伙伴，不是医生，不能替代专业帮助" |
| **验收标准** | 1. 安全规则在 100% 的对话中生效；2. "紧急帮助"按钮在对话界面始终可见且可点击；3. 安全声明在用户首次对话中出现 |
| **优先级** | P0 |

#### FR-3.5 对话内容加密

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Ubiquitous |
| **需求描述** | The system shall 对所有星宝对话内容进行端到端加密存储（AES-256-GCM），加密密钥由用户设备端管理，服务端无法解密对话内容 |
| **验收标准** | 1. 对话数据在服务端以密文形式存储；2. 仅持有密钥的客户端可解密读取；3. 通过安全审计验证 |
| **优先级** | P0 |

### 4.4 模块四：情绪探索

#### FR-4.1 基础情绪测评

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Event-driven |
| **需求描述** | When 用户进入"探索"页面并选择"情绪探索"，the system shall 展示一个去标签化的心理健康评估问卷，底层映射 PHQ-9 量表（9 题），前端呈现为轻松对话式问答（如"最近两周，你感觉心情低落的频率是？"），结果不输出诊断结论，而是以温暖的语言描述当前情绪状态，并提供个性化建议 |
| **验收标准** | 1. 完成测评不超过 3 分钟；2. 结果页不包含任何诊断性标签（如"中度抑郁"）；3. 结果页包含 3 条以上个性化建议；4. 量表信效度与 PHQ-9 标准版相关系数 > 0.85 |
| **优先级** | P0 |

#### FR-4.2 测评结果处理

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Event-driven |
| **需求描述** | When 测评结果显示用户可能存在中度以上困扰（PHQ-9 ≥ 10），the system shall 在结果页温和建议用户多与星宝聊聊或考虑寻求专业帮助，同时推荐相关的冥想/放松内容。If 结果显示高风险（PHQ-9 ≥ 20 或第 9 题自伤意念 ≥ 1），then the system shall 触发危机干预流程（见 FR-5.1） |
| **验收标准** | 1. 中度以上自动推荐至少 1 条相关放松内容；2. 高风险触发率 100%；3. 建议语言为邀请式而非指令式 |
| **优先级** | P0 |

### 4.5 模块五：冥想放松

#### FR-5.1 冥想音频播放

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Event-driven |
| **需求描述** | When 用户选择冥想内容并点击播放，the system shall 播放冥想音频，提供播放控制（播放/暂停、前进/后退 15 秒、音量调节），支持后台播放和锁屏播放。冥想内容时长控制在 3-10 分钟 |
| **验收标准** | 1. 音频播放延迟 < 2 秒；2. 支持后台播放和锁屏控制；3. 播放进度自动保存 |
| **优先级** | P1 |

#### FR-5.2 MVP 冥想内容清单

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Optional |
| **需求描述** | Where MVP 阶段，the system shall 提供 3-5 个冥想/放松音频，覆盖以下主题：①考前放松（5 分钟）；②入睡困难（8 分钟）；③情绪风暴安抚（5 分钟）；④专注力提升（3 分钟）；⑤身体扫描放松（10 分钟） |
| **验收标准** | 1. 至少提供 3 个冥想音频；2. 每个音频有明确的主题标签和时长标识；3. 音频内容经心理健康专业人士审核 |
| **优先级** | P1 |

#### FR-5.3 呼吸练习

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Event-driven |
| **需求描述** | When 用户进入呼吸练习，the system shall 以动画可视化引导呼吸节奏，支持至少 2 种呼吸模式（4-7-8 呼吸法、方块呼吸法），动画元素与品牌"星球"主题一致 |
| **验收标准** | 1. 呼吸动画流畅（60fps）；2. 呼吸节奏与动画同步；3. 练习可设置 1/3/5 分钟时长 |
| **优先级** | P1 |

### 4.6 模块六：风险检测与危机响应

#### FR-6.1 关键词风险检测（L1）

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Ubiquitous |
| **需求描述** | The system shall 对用户与星宝的对话内容进行实时关键词匹配，检测明确的自伤/自杀/暴力相关关键词。If 命中关键词，then the system shall 立即触发危机响应流程（FR-6.3） |
| **验收标准** | 1. 关键词检测延迟 < 500ms；2. 关键词库覆盖率经心理专家审核；3. 误报率 < 5% |
| **优先级** | P0 |

#### FR-6.2 语义风险检测（L2）

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Ubiquitous |
| **需求描述** | The system shall 基于 NLP 语义理解模型对用户对话内容进行意图分析，识别隐含的自伤/自杀意念（即使未触发 L1 关键词），并综合用户心情打卡历史和测评结果计算综合风险等级（绿/黄/橙/红） |
| **验收标准** | 1. 语义分析在每次对话后异步完成，延迟 < 5 秒；2. 风险等级每日更新；3. L2 召回率 > 85%（相比仅用 L1） |
| **优先级** | P0 |

#### FR-6.3 高风险危机响应

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Event-driven |

| **优先级** | P0 |

#### FR-6.4 橙色风险主动干预

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Event-driven |
| **需求描述** | When 系统判定用户为橙色风险等级（中度困扰，持续情绪低落），the system shall 通过星宝发起主动关怀对话，推荐情绪探索测评和放松内容，并温和建议"有时候和人聊聊会有帮助"（MVP 阶段不包含人工转介，仅做引导） |

## 6. 核心用户流程

- 用户在注册流程中途退出 → 下次打开从退出点继续

```
打开APP → 首页

    ├── [路径B] 点击"和星宝聊聊" → 对话界面 → 自由聊天 / 选话题卡片
    └── [路径C] 探索页 → 冥想放松 / 情绪探索测评
```

| API 响应时间 | < 500ms（P95） |
| AI 对话首字延迟 | < 1.5 秒 |
| 崩溃率 | < 0.1% |
| 安装包大小 | < 50MB（Android）/ < 80MB（iOS） |

### 9.3 兼容性验收标准

| 平台 | 最低版本 | 覆盖 |
|------|---------|------|
| iOS | iOS 15+ | iPhone 8 及以上 |
| Android | Android 9+（API 28+） | 主流品牌（华为/小米/OPPO/vivo/三星） |

### 9.4 安全合规验收

| 标准 | 要求 |
|------|------|
| 数据传输 | HTTPS/TLS 1.3 全程加密 |
| 对话存储 | AES-256-GCM 端到端加密 |

| 数据删除权 | 用户可一键删除账号及全部数据 |
| 第三方 SDK | 列出所有 SDK 及数据收集说明 |

## 10. 附录

### 10.1 非功能性需求

| 类别 | 需求 |
|------|------|
| **可用性** | 界面文案符合 12 岁阅读水平（无专业术语）；关键操作提供视觉/触觉/听觉反馈 |
| **可访问性** | 支持系统字体缩放（最小 1.2x 不失布局）；关键元素有足够的触控区域（≥ 44pt） |
| **国际化** | MVP 仅支持简体中文，预留 i18n 框架 |
| **离线能力** | 心情打卡支持离线操作，联网后同步 |
| **电池与流量** | 后台播放功耗优化；冥想音频支持 WiFi 预加载 |

### 10.2 MVP 不做清单（明确边界）

以下功能**明确不在 MVP 范围**，避免范围蔓延：

- ❌ 咨询师匹配、预约、咨询功能
- ❌ 家长端 APP/小程序
- ❌ 学校管理端 Web 后台
- ❌ 匿名树洞社区
- ❌ CBT 自助练习库（思维记录、行为激活等结构化练习）
- ❌ 游戏化成长体系（星际任务、成就徽章、星球装饰）
- ❌ 微信小程序
- ❌ 情绪周报/月报/学期报告
- ❌ 语音/涂鸦日记
- ❌ 解压小工具（指尖陀螺、涂色板等）
- ❌ 人工回访机制
- ❌ 内容推荐算法
