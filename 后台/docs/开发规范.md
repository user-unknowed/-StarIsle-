# 开发规范 - 星屿 StarIsle MVP v1.0

## 代码规范

### 1. Flutter前端规范

#### 目录结构
```
frontend/
├── lib/
│   ├── main.dart           # 应用入口
│   ├── src/
│   │   ├── app.dart        # 应用配置
│   │   ├── screens/        # 页面组件
│   │   ├── widgets/        # UI组件
│   │   ├── providers/      # 状态管理
│   │   ├── services/       # API服务
│   │   ├── models/         # 数据模型
│   │   ├── utils/          # 工具类
│   │   └── theme/          # 主题配置
│   └── assets/             # 静态资源
└── pubspec.yaml            # 依赖配置
```

#### 代码风格
- 使用Riverpod进行状态管理
- 每个组件单独文件
- 遵循Material Design 3设计规范
- 使用const构造器优化性能

#### 命名规范
- 文件名：小写+下划线 `home_screen.dart`
- 类名：大驼峰 `HomeScreen`
- 变量名：小驼峰 `userName`
- 常量名：小驼峰 `primaryColor`

#### 命名规范

- 文件名：小驼峰

- 结构体名：大驼峰

## Git提交规范

### Commit Message格式
```
<type>(<scope>): <subject>

<body>

<footer>
```

### Type类型
- feat: 新功能
- fix: Bug修复
- docs: 文档更新
- style: 代码格式调整
- refactor: 重构
- test: 测试相关
- chore: 构建/工具相关

### 示例
```
feat(chat): 新增WebSocket实时对话功能

- 实现WebSocket连接管理
- 添加消息加密处理
- 支持实时风险检测

Closes #123
```

## 测试规范

### 1. 单元测试
- 每个模块必须有单元测试
- 测试覆盖率 > 80%
- 使用Mock对象隔离依赖

### 2. 集成测试
- 测试API端到端流程
- 验证数据库交互
- 测试外部服务调用

### 3. 性能测试
- API响应时间测试
- 并发压力测试
- 内存泄漏检测

## 安全规范

### 1. 代码安全
- 不硬编码密钥和密码
- 使用环境变量存储敏感信息
- 定期更新依赖包，修复安全漏洞

### 2. 数据安全
- 对话内容必须加密存储
- 用户数据最小化采集
- 实现数据访问权限控制

### 3. API安全
- 实现JWT认证
- 配置速率限制
- 输入参数验证
- SQL注入防护

## 文档规范

### 1. API文档
- 使用Swagger/OpenAPI规范
- 每个接口必须有详细说明
- 包含请求/响应示例

### 2. 代码注释
- 公共函数必须有文档注释
- 复杂逻辑必须添加解释注释
- 避免无意义注释

### 3. README文档
- 项目概述
- 快速开始指南
- 环境配置说明
- 常见问题解答

## 版本管理

### 版本号规范
- 遵循语义化版本控制（Semantic Versioning）
- 格式：`MAJOR.MINOR.PATCH`
- MVP版本：`v1.0.0`

### 发布流程
1. 代码审查
2. 测试验证
3. 文档更新
4. 版本发布
5. 部署上线
