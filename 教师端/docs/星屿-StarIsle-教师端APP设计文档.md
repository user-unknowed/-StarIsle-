## 1. 产品定位与角色体系

### 1.1 产品定位

> **为班主任、科任老师与心理老师打造的「青少年心理守护协同工作台」——让一线教师能轻松识别、安全上报、专业介入，让学生在最需要时被看见。**

一句话描述：**让每一位老师都成为学生心理安全网的「第一响应者」与「专业接棒人」。**

### 1.2 与学生端的关系

| 维度 | 学生端 | 教师端 |
|------|--------|--------|
| **角色** | 被陪伴者、求助者 | 守护者、协同者、介入者 |
| **核心体验** | 温暖、零压力、去标签 | 专业、高效、有据可依 |
| **品牌调性** | "懂你的学长学姐" | "专业又有人情味的搭档" |
| **语言风格** | 萌系小精灵风（小星） | 去标签化但更务实 |
| **视觉** | 星夜蓝 + 暖橙，圆润治愈 | 同色系，信息密度更高、状态色更鲜明（绿/黄/橙/红风险色保留） |
| **共有元素** | 星球旋转加载、星光粒子、星空主题 | 继承品牌资产，插画更克制 |

### 1.3 品牌调性原则（教师端专属）

1. **专业化但不冷冰冰**：保留星屿温暖基因，以"靠谱搭档"姿态出现
2. **去标签化延续**：对学生用"情绪困扰/状态不太稳定"等中性表达；内部沟通可用"风险等级/症状反馈"等专业判断
3. **减负优先**：所有流程以"老师已经很忙"为前提，3 步内能完成的绝不设计 5 步
4. **安全兜底**：高风险场景系统主动兜底，不让老师独自承担判断压力

### 1.4 两种角色权限对比

| 维度 | 班主任/科任老师（观察者） | 心理老师（专业介入者） |
|------|------|------|
| **核心职责** | 日常观察、症状上报、自我求助 | 接收报告、观察对话、专业介入、个案管理 |
| **学生症状反馈** | ✅ 可发起 | ✅ 接收并处理 |
| **教师自我求助** | ✅ 可反馈自身状态 | ✅ 心理老师间可互相督导 |
| **学生-AI 对话观察** | ❌ 不可查看 | ✅ 高风险自动开放 / 其他需授权 |
| **对话介入干预** | ❌ 不可介入 | ✅ 可替代 AI 直接与学生对话 |
| **学生档案查看** | ⚠️ 仅自己上报记录与回执 | ✅ 经授权/高风险可查看 |
| **解压小游戏** | ✅ | ✅ |
| **知识库** | ⚠️ 基础版 | ✅ 完整版（含干预手册、危机SOP、督导社区） |
| **班级情绪概览** | ✅ 本班匿名聚合（≥30人脱敏） | ✅ 负责学生群体 + 跨班级高风险看板 |
| **看到学生真实身份** | ✅（本班学生） | ⚠️ 仅高风险/已授权学生 |

## 3. 信息架构与 Tab 设计

### 3.1 Tab 结构（与学生端 4 Tab 一致）

| 学生端 Tab | 教师端 Tab | 定位 |
|-----------|-----------|------|
| 首页 | **工作台** | 今日概览、待办、告警、快捷入口 |
| 聊天 | **学生** | 学生档案、症状反馈、对话观察（心理老师） |

## 4. 四大核心功能详细设计

> EARS 类型：Event-driven（事件驱动）/ State-driven（状态驱动）/ Ubiquitous（常态）/ Optional（可选）

### 4.1 功能一：学生症状反馈与报告

#### FR-T1.1 症状反馈表填写

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Event-driven |
| **需求描述** | When 班主任/科任老师点击「上报学生情况」，the system shall 展示症状反馈表，引导教师结构化记录观察信息。表单字段包括：①学生选择（本班学生列表）；②观察到的行为表现（多选：情绪低落/易怒/社交退缩/学业骤降/睡眠异常/自伤痕迹/其他）；③情绪表现（多选：哭泣/麻木/焦虑/暴躁/异常平静）；④持续时间（单选：<1周/1-2周/2-4周/>1月）；⑤严重程度（单选：轻微关注/需要关注/较为紧急/紧急，对应颜色提示）；⑥具体描述（文本，选填但强烈建议）；⑦是否已与学生沟通（单选）；⑧是否已联系家长（单选）。提交前系统根据严重程度给出分级响应建议提示 |
| **验收标准** | 1. 表单完成不超过 3 分钟；2. 行为/情绪/持续时间/严重程度为必填；3. 严重程度选择"较为紧急/紧急"时系统强提示并建议立即联系心理老师；4. 提交后有明确成功反馈与流转状态 |
| **优先级** | P0 |

#### FR-T1.2 报告流转与通知

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Event-driven |
| **需求描述** | When 班主任提交症状反馈报告，the system shall 自动将报告路由至对应年级/班级的心理老师，并按严重程度分级推送：①"紧急"级别（红色）→ 心理老师 APP 强提醒 + 短信通知 + 工作台置顶告警，要求 2 小时内响应；②"较为紧急"（橙色）→ APP 推送 + 工作台告警，要求 24 小时内响应；③"需要关注"（黄色）→ APP 推送，要求 48 小时内响应；④"轻微关注"（绿色）→ 工作台待办列表，72 小时内查看。报告流转全程状态可追踪：已提交→已接收→处理中→已处理 |
| **验收标准** | 1. 紧急报告推送延迟 < 1 分钟；2. 心理老师超时未响应时系统自动升级提醒（向上级心理组长脱敏告警）；3. 班主任可实时查看处理状态与回执 |
| **优先级** | P0 |

#### FR-T1.3 心理老师接收与处理

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Event-driven |
| **需求描述** | When 心理老师打开待处理报告，the system shall 展示完整报告内容，并提供处理操作：①标记处理状态（已查看/跟进中/已处理）；②填写处理意见（初步判断/已采取措施/后续建议）；③一键关联学生档案；④如需深度介入，一键发起对话观察或预约面谈；⑤处理后向班主任回执（脱敏，不含咨询细节，仅含"已接收并跟进"与处理结论摘要）。处理记录归档至学生档案 |
| **验收标准** | 1. 回执内容经隐私脱敏；2. 处理记录自动归档且不可删除；3. 涉及高风险时强制要求填写跟进计划 |
| **优先级** | P0 |

#### FR-T1.4 行为观察速记

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Optional |
| **需求描述** | Where 教师日常观察到学生异常但尚未达到上报程度，the system shall 提供「观察速记」功能，支持教师快速记录单条观察（学生+行为标签+一句话描述），积累后可一键汇总成正式反馈报告 |
| **验收标准** | 1. 速记可在 30 秒内完成；2. 速记仅自己可见，不触发流转；3. 支持多条速记合并为正式报告 |
| **优先级** | P1 |

### 4.2 功能二：教师自我心理状态反馈

#### FR-T2.1 教师心情打卡

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Event-driven |
| **需求描述** | When 教师点击心情打卡，the system shall 展示 5 档心情选择（与学生端一致的星球表情），教师 1 次点击完成打卡，可选填压力来源标签（教学任务/学生问题/家校沟通/评价考核/个人生活/其他）。打卡数据仅用于教师个人趋势查看，对学校管理层完全不可见 |
| **验收标准** | 1. 打卡 ≤ 3 次点击；2. 打卡数据存储于独立加密空间，管理层账号无任何访问权限；3. 个人可查看 7/30 日心情趋势 |
| **优先级** | P0 |

#### FR-T2.2 自我状态反馈与求助

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Event-driven |
| **需求描述** | When 教师选择「我要找人聊聊」发起自我求助，the system shall 引导教师填写状态反馈表（当前困扰描述/希望获得的支持类型：倾听/建议/正式咨询/紧急支持/紧急程度），并匹配至非本年级的心理老师（避免利益冲突），通过私密对话通道建立连接。求助记录存储于独立隔离空间，仅求助教师与对接心理老师可见 |
| **验收标准** | 1. 求助对接延迟 < 4 小时（非紧急）/ 立即响应（紧急）；2. 求助记录对学校管理层、其他班主任、本年级心理老师完全不可见；3. 对话端到端加密 |
| **优先级** | P0 |

#### FR-T2.3 教师求助隐私隔离

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Ubiquitous |
| **需求描述** | The system shall 对教师自我求助记录实施严格的隐私隔离：①求助记录与学生数据物理隔离存储（独立数据库表 + 独立加密密钥）；②学校管理端、教育局后台、其他教师账号均无任何访问接口；③仅求助教师本人与对接心理老师可见；④对接心理老师签署额外保密协议后方可查看；⑤求助记录不纳入教师考核/档案系统；⑥求助教师可一键删除自己的求助记录 |
| **验收标准** | 1. 通过权限审计验证管理层账号零访问；2. 求助记录删除后不可恢复；3. 隐私隔离机制通过独立安全审计 |
| **优先级** | P0 |

### 4.3 功能三：学生-AI 对话观察与介入（心理老师专属）

#### FR-T3.1 对话查看权限矩阵

| 属性 | 内容 |
|------|------|
| **EARS 类型** | State-driven |
| **需求描述** | While 心理老师查看学生对话，the system shall 按以下权限矩阵控制访问：①学生风险等级为橙色/红色时，自动开放对话查看权限给该学生绑定的心理老师（学生注册时已知情同意此机制）；②风险等级为绿色/黄色时，需学生主动授权（学生端发起授权请求，心理老师接受后查看）；③心理老师仅能查看自己负责的学生；④所有对话查看行为记录审计日志（谁、何时、查看何学生）；⑤查看界面默认脱敏显示，需主动点击"查看原文"才展示完整内容并二次记录 |
| **验收标准** | 1. 橙/红风险触发后心理老师可见延迟 < 5 分钟；2. 权限校验在每次访问时执行；3. 审计日志不可篡改 |
| **优先级** | P0 |

#### FR-T3.2 实时观察模式

| 属性 | 内容 |
|------|------|
| **EARS 类型** | State-driven |
| **需求描述** | While 心理老师进入某学生的对话观察页，the system shall 以只读流式展示学生与小星的实时对话（WebSocket 推送新消息），并提供：①对话风险等级实时标注（每条消息旁标注风险色）；②小星的对话策略提示（如"当前为共情回应""CBT 认知重构引导"）；③AI 风险检测告警（当系统检测到风险升级时在观察页显著提示）；④「介入对话」按钮（满足介入条件时高亮）；⑤观察计时与笔记区（心理老师可边看边记） |
| **验收标准** | 1. 实时消息延迟 < 3 秒；2. 风险标注与 AI 检测同步；3. 观察笔记自动归档至学生档案 |
| **优先级** | P0 |

#### FR-T3.3 介入干预模式

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Event-driven |
| **需求描述** | When 心理老师点击「介入对话」并确认介入，the system shall 切换对话模式：①学生侧对话界面显示"心理老师加入对话，小星先退到一旁啦~"（温暖过渡，不打断学生）；②心理老师以真实身份（"XX 学校心理老师"）直接回复学生，替代小星进行专业干预；③介入期间小星暂停自动回复，但保持在线状态；④介入对话内容与学生原对话连贯存储，但标注"心理老师介入"消息；⑤介入期间系统提供话术辅助（危机干预话术库、CBT 引导话术参考）；⑥心理老师可随时「结束介入」，小星恢复接管，并向学生温和说明"老师先去忙啦，小星继续陪你~" |
| **验收标准** | 1. 介入切换延迟 < 2 秒；2. 学生侧过渡话术温暖无突兀感；3. 介入消息明确标识身份 |
| **优先级** | P0 |

#### FR-T3.4 介入记录与归档

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Event-driven |
| **需求描述** | When 心理老师结束介入，the system shall 自动生成介入记录并归档至学生档案，内容包括：①介入起止时间；②介入轮次与消息数；③心理老师填写的介入小结（介入判断/采取策略/学生反应/后续建议，SOAP 简版）；④风险等级变化（介入前 vs 介入后）；⑤是否需要后续跟进（单选 + 跟进计划）。介入记录对班主任回执中仅显示"心理老师已介入跟进"，不暴露介入细节 |
| **验收标准** | 1. 介入记录自动归档不可删除；2. 介入小结为必填；3. 班主任回执脱敏 |
| **优先级** | P0 |

#### FR-T3.5 学生授权管理

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Event-driven |
| **需求描述** | When 学生在学生端发起对话授权请求（针对绿/黄风险情况），the system shall 将请求推送给指定心理老师，心理老师可接受/拒绝。授权需明确范围（仅查看最近 7 天 / 持续观察 30 天）与可撤销。授权状态在学生档案中可见 |
| **验收标准** | 1. 授权请求推送 < 1 分钟；2. 授权到期自动失效；3. 学生可随时撤销授权 |
| **优先级** | P1 |

### 4.4 功能四：解压小游戏

#### FR-T4.1 舒压空间入口与游戏清单

| 属性 | 内容 |
|------|------|
| **EARS 类型** | Optional |
| **需求描述** | Where 教师在「我的 → 舒压空间」或工作台「舒压一下」入口，the system shall 提供 5 个轻量休闲小游戏，每个游戏单局 1-3 分钟，无需登录进度，即开即玩，退出即止。游戏清单：①捏泡泡纸；②指尖陀螺；③涂色板；④白噪音生成器；⑤星空呼吸 |
| **验收标准** | 1. 游戏加载 < 2 秒；2. 无强制注册/计分压力；3. 支持后台音频（白噪音） |
| **优先级** | P1 |

#### 各游戏玩法设计

**游戏 1：捏泡泡纸**
- 玩法：屏幕呈现一张泡泡纸，点击泡泡即"啵"一声破裂并出现星光粒子，可逐个捏或滑动连捏。整张捏完自动翻面，无限循环
- 解压点：触觉反馈 + 满足感 + ASMR 音效
- 细节：泡泡破裂音效可选（经典/气泡/水滴），支持震动反馈

**游戏 2：指尖陀螺**
- 玩法：屏幕中央一个星球主题陀螺，滑动即可旋转，旋转速度随滑动力度变化。陀螺旋转时星光粒子环绕，逐渐减速至停止
- 解压点：惯性物理 + 视觉流畅 + 无目标无压力
- 细节：可切换 3 种陀螺皮肤（星球/星云/彗星），记录最长旋转时间（仅本地）

**游戏 3：涂色板**
- 玩法：提供 8-12 幅星球/星空主题线稿，教师选择颜色填充，可缩放细节涂色。完成可保存为壁纸
- 解压点：心流沉浸 + 创造表达 + 成果留存
- 细节：调色板含星屿品牌色系，支持撤销/重做，自动保存进度

**游戏 4：白噪音生成器**
- 玩法：提供多个音源混音器（雨声/海浪/篝火/风铃/键盘/图书馆白噪），各音源独立音量调节，可组合自定义白噪音方案。支持后台播放、定时关闭
- 解压点：听觉放松 + 个性化 + 助眠/助专注
- 细节：预设 4 种组合方案（专注/放松/入睡/冥想），支持保存自定义方案

**游戏 5：星空呼吸**
- 玩法：复用学生端呼吸练习的可视化（星球缩放），提供 4-7-8 呼吸法与方块呼吸法，引导教师 1-5 分钟呼吸放松
- 解压点：生理调节 + 视觉引导 + 即时平静
- 细节：与教师心情打卡联动（打卡后推荐呼吸）

#### FR-T4.2 舒压引导

| 属性 | 内容 |
|------|------|
| **EARS 类型** | State-driven |
| **需求描述** | While 教师连续 3 天心情打卡为"不太好/很糟"，或刚完成一次高强度上报/介入，the system shall 在工作台温和推荐舒压空间（"今天辛苦啦，要不要捏捏泡泡放松一下？"），非强制、可关闭 |
| **验收标准** | 1. 推荐为非阻断式轻提示；2. 可在设置中关闭 |
| **优先级** | P2 |

```

        │

        ▼
```

## 8. 数据安全与隐私

### 8.1 学生隐私保护

| 维度 | 保护措施 |
|------|---------|
| **对话查看权限** | 严格按权限矩阵：橙红自动开放给绑定心理老师，绿黄需学生授权；班主任无对话查看权 |
| **对话内容脱敏** | 心理老师查看时默认脱敏视图，需主动"查看原文"并二次审计留痕 |
| **档案访问控制** | 心理老师仅能访问高风险/已授权学生；班主任仅见本班匿名聚合（≥30人脱敏）+ 自己的上报记录与回执 |
| **回执脱敏** | 班主任收到的处理回执仅含"已接收并跟进"+ 处理结论摘要，不含咨询/介入细节 |
| **端到端加密** | 复用学生端 AES-256-GCM 加密体系，对话内容服务端零知识 |
| **审计日志** | 所有人对学生数据的访问行为全程留痕，不可篡改，学生可在隐私中心查看"谁访问了我的数据" |
| **最小聚合原则** | 班级/群体情绪概览最小聚合单元 ≥ 30 人，防止推断个体 |

### 8.2 教师隐私保护

| 维度 | 保护措施 |
|------|---------|
| **求助记录物理隔离** | 教师自我求助记录与学生数据独立数据库表 + 独立加密密钥存储 |
| **管理层零访问** | 学校管理端、教育局后台、其他教师账号均无任何访问接口 |
| **对接心理老师限制** | 匹配非本年级心理老师（避免利益冲突）；对接心理老师签署额外保密协议 |
| **不计入考核** | 求助记录不纳入教师考核/档案系统 |
| **可删除** | 教师可一键删除自己的求助记录，删除后不可恢复 |
| **心情打卡私有** | 教师心情打卡数据仅自己可见，管理层无访问权 |

### 8.3 数据访问审计

| 审计对象 | 记录内容 | 可见方 |
|---------|---------|--------|
| 学生对话查看 | 心理老师ID、学生ID、查看时间、是否查看原文 | 学生本人（隐私中心）+ 安全审计员 |
| 学生档案访问 | 访问者ID、学生ID、访问字段、时间 | 学生本人 + 安全审计员 |
| 教师求助记录访问 | 访问者ID、求助教师ID、时间 | 仅限求助教师与对接心理老师 |
| 症状反馈报告流转 | 报告ID、发起者、接收者、状态变更、时间 | 报告相关方 |
| 介入记录 | 心理老师ID、学生ID、介入时间 | 心理老师 + 学生档案 |

审计日志采用 **append-only（仅追加）** 存储，不可篡改，定期由独立安全审计员审查异常访问。

### 9.3 与学生端数据流关系

```
┌─────────────┐                    ┌─────────────┐
│  学生端 APP   │                    │  教师端 APP   │
├─────────────┤                    ├─────────────┤
│ ·心情打卡    │──┐                │ ·工作台告警   │
│ ·小星对话    │  │ 风险等级        │ ·症状反馈     │
│ ·情绪探索    │──┼────────────────→│ ·对话观察     │
│ ·风险触发    │  │ (橙红自动开放)  │ ·对话介入     │
└─────────────┘  │                │ ·自我求助     │
        ↑        │                │ ·解压游戏     │
        │        │                └──────┬──────┘
        │        │                       │
        │   ┌────┴────────────────────────┴────┐
        │   │          后端服务层                 │
        │   ├──────────────────────────────────┤
        │   │ 用户/对话/测评/风险/通知 (复用)     │
        │   │ + 教师/反馈/求助/观察/介入/授权(新增)│
        │   └──────────────────────────────────┘
        │                       │
        └─── 介入消息回流 ←──────┘
       (心理老师介入时，
        消息通过对话服务路由回学生端)
```

**关键数据流**：
1. **风险告警流**：学生端风险检测 → 风险检测服务 → 通知服务 → 教师端告警 + 对话观察权限自动开放
2. **症状反馈流**：教师端发起 → 症状反馈服务 → 路由至心理老师 → 处理回执回流至班主任
3. **对话介入流**：心理老师介入 → 对话介入服务 → 对话服务路由 → 学生端显示心理老师消息
4. **自我求助流**：教师端发起 → 教师求助服务（隔离存储）→ 匹配心理老师 → 私密对话通道
5. **授权流**：学生端发起授权 → 授权管理服务 → 心理老师端处理 → 对话观察权限生效

## 10. 附录：实施建议与风险

### 10.1 阶段划分建议

| 阶段 | 范围 | 说明 |
|------|------|------|
| **Phase 1** | 角色体系 + 症状反馈 + 工作台告警 + 知识库基础 | 打通班主任→心理老师上报闭环 |

| **Phase 3** | 教师自我求助 + 隐私隔离 + 班级聚合概览 | 教师自助与群体洞察 |

### 10.2 关键风险

| 风险 | 等级 | 应对 |
|------|:---:|------|

| 对话介入时学生体验突兀 | 🟠 高 | 温暖过渡话术设计 + 灰度测试学生接受度 |
| 班主任上报负担过重导致弃用 | 🟠 高 | 速记轻量化 + 3 分钟内完成 + 预填充 |
| 心理老师介入响应超时 | 🟠 高 | 分级时效 + 超时自动升级 + 工作量监控 |
| 权限矩阵复杂导致越权 | 🟠 高 | 每次访问实时校验 + 审计日志 + 渗透测试 |
