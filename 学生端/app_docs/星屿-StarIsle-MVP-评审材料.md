## 2. 需求背景速览

### 2.1 产品一句话

> 「星屿」是为 12-18 岁初高中生打造的 AI 情绪成长伙伴 APP，通过极简心情打卡和 24/7 AI 对话，零压力提供第一时间情绪支持。

### 2.2 MVP 范围（7 模块 / 18 页面 / 23 条需求）

| 模块 | 功能 | 页面数 | 优先级分布 |
|------|------|--------|-----------|
| 注册与隐私 | 匿名注册、隐私说明、隐私设置 | 3 | P0×3 |
| 心情打卡 | 极简打卡、晴雨表、智能追问、连续激励 | 1（首页内） | P0×3, P1×1 |
| AI 星宝对话 | 共情对话、话题卡片、沉默关怀、安全边界、加密 | 2 | P0×3, P1×2 |
| 情绪探索 | 对话式 PHQ-9 测评、结果处理 | 2 | P0×2 |
| 冥想放松 | 音频播放、呼吸练习 | 3 | P1×3 |
| 风险检测与危机响应 | L1 关键词 + L2 语义检测、红/橙风险响应 | 1（危机资源页） | P0×3, P1×1 |
| 帮助中心 | 危机资源、使用指南 | 2 | P0×1, P1×1 |

### 2.3 核心指标

| 指标 | MVP 目标 |
|------|---------|
| 7 日留存率 | > 30% |
| 30 日留存率 | > 15% |
| 心情打卡率 | > 40% |
| 用户 NPS | > 40 |
| 高风险热线触达率 | > 80% |

### 3.3 设计关键决策点（待确认）

| 编号 | 决策点 | 选项 | 建议 |
|------|--------|------|------|
| D-D01 | **品牌动画风格** | A) 2D 逐帧动画（温暖手绘风）/ B) Lottie 矢量动画（科技感） | **建议 A**，更契合青少年温暖调性 |
| D-D02 | **心情表情风格** | A) Emoji 原生 / B) 自定义星球主题表情 | **建议 B**，增强品牌辨识度 |
| D-D03 | **暗色模式策略** | A) 跟随系统 / B) APP 内独立开关 / C) 两者都支持 | **建议 C**，给用户最大灵活度 |
| D-D04 | **隐私说明形式** | A) 静态漫画 / B) 微动画漫画 / C) 短视频 | **建议 B**，成本可控且生动 |
| D-D05 | **冥想页星球动画** | A) 2D 呼吸缩放 / B) 3D 自转星球 / C) 粒子效果 | **建议 A**，性能友好且符合 MVP 成本 |
| D-D06 | **Tab Bar 样式** | A) 文字+图标 / B) 仅图标 / C) 浮动胶囊式 | **建议 A**，青少年需要明确导航 |

### 3.4 设计验收关键点

1. **去标签化**：全界面不出现"抑郁""焦虑""治疗""诊断"等词汇
2. **色彩体系**：主色调为星夜蓝 + 暖橙渐变，避免冷冰冰的医疗感
3. **字体层级**：正文 ≥ 15sp（青少年阅读舒适），标题清晰区分
4. **触控区域**：所有可点击元素 ≥ 44pt（iOS）/ 48dp（Android）
5. **无障碍**：关键图标有文字标签，支持系统字体缩放
6. **品牌一致性**：星球/星光/星空主题元素贯穿全流程

### 5.2 数据库设计要点

| 数据域 | 存储方案 | 关键字段 | 加密要求 |
|--------|---------|---------|---------|
| 用户身份 | PostgreSQL | `user_id`, `nickname`, `avatar`, `age_group`, `created_at` | 昵称明文，user_id 为 UUID |
| 心情打卡 | PostgreSQL | `user_id`, `date`, `mood_level(1-5)`, `tags[]` | 关联 user_id |

| 风险记录 | PostgreSQL | `user_id`, `risk_level`, `trigger_source`, `detected_at`, `action_taken` | 严格访问控制 |
| 密钥管理 | HSM/KMS | `user_id`, `public_key`, `encrypted_private_key` | HSM 硬件保护 |

### 5.3 后端技术风险

| 风险 | 等级 | 应对 |
|------|------|------|
| LLM 对话延迟影响用户体验 | 🔴 极高 | 流式输出（SSE/WebSocket）+ 模型量化 + GPU 推理优化 |
| 端到端加密增加系统复杂度 | 🟠 高 | 明确密钥交换协议，充分测试加解密流程 |
| MongoDB 对话量增长快 | 🟡 中 | TTL 索引自动归档 + 冷热数据分离 |
| 高并发下 WebSocket 连接数压力 | 🟡 中 | 水平扩展 + 连接数监控 + 自动扩缩容 |
| 风险检测漏报 | 🔴 极高 | 多层检测 + 定期人工抽检 + 关键词库持续更新 |

## 6. AI 研发视角

### 6.1 AI 模型需求

| 模型 | 用途 | 技术方案 | 训练数据需求 |
|------|------|---------|------------|
| **星宝对话模型** | 共情 + CBT 对话 | 国产大模型微调（Qwen/DeepSeek） | 青少年心理咨询对话语料（需脱敏） |
| **情绪分析模型** | 对话/日记情绪分类 | BERT-based 文本分类 | 标注情绪标签的青少年文本 |
| **风险检测模型（L2）** | 语义级自伤/自杀意图识别 | 意图分类 + 上下文建模 | 标注风险等级的对话样本 |
| **内容安全模型** | 有害内容过滤 | 规则 + 分类模型 | 自伤方法/欺凌/诱导性内容样本 |

### 6.2 AI 关键约束

| 约束 | 要求 |
|------|------|
| **对话安全** | 绝不输出自伤/自杀方法；不输出诊断结论；不替代医生 |
| **共情优先** | 每次回复先回应情绪，再提供引导 |
| **去标签化** | 不使用"抑郁""焦虑症""治疗"等诊断性词汇 |
| **CBT 框架** | 对话遵循 CBT 原则：识别自动思维 → 挑战认知扭曲 → 替代思维 |
| **生成式 AI 备案** | 大模型服务需通过国家生成式 AI 服务备案 |
| **内容标识** | AI 生成内容需明确标识（"星宝是 AI 伙伴"） |

### 6.3 AI 待确认问题

| 编号 | 问题 | 重要性 |
|------|------|--------|
| AI-Q01 | 使用哪个国产大模型作为基座？（Qwen / DeepSeek / 其他） | 🔴 高 |
| AI-Q02 | 模型部署方式？（私有化 / 云 API / 混合） | 🔴 高 |
| AI-Q03 | 微调语料来源？是否有现成的青少年心理咨询对话数据？ | 🟠 高 |
| AI-Q04 | CBT 对话质量如何评估？谁来做人工评审？ | 🟠 高 |
| AI-Q05 | 风险检测模型的训练样本如何获取？ | 🟠 高 |
| AI-Q06 | 内容安全是否需要接入第三方审核 API（如网易易盾）？ | 🟡 中 |

| 测试类型 | 范围 | 工具 |
