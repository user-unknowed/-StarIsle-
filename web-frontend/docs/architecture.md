# 星屿心理健康管理系统 - 跨平台前端架构设计文档

## 1. 概述

### 1.1 项目背景
星屿心理健康管理系统是一个面向青少年心理健康的综合性管理平台，支持学生、教师和家长三种角色的使用。本项目旨在通过Web技术构建一个跨平台的前端框架，实现安卓、iOS和鸿蒙三端的统一用户体验。

### 1.2 技术目标
- 构建基于React的跨平台前端框架
- 实现三端统一的设计规范和交互体验
- 支持响应式布局，适配不同屏幕尺寸
- 确保高性能和流畅的用户体验

## 3. 核心模块设计

### 3.1 设计系统

#### 3.1.1 设计令牌 (Design Tokens)
设计令牌是设计系统的核心，定义了可复用的设计属性：

- **颜色系统**: 包含主色、次色、成功、警告、危险等颜色
- **间距系统**: 定义8种标准间距（xs到4xl）
- **字体系统**: 定义字体大小、字重、行高和字体族
- **阴影系统**: 定义6种阴影级别
- **圆角系统**: 定义7种圆角大小
- **过渡系统**: 定义过渡时长和缓动函数
- **断点系统**: 定义6种响应式断点

#### 3.1.2 主题配置
支持浅色和深色两种主题模式，通过CSS类名切换：
- `light`: 浅色主题（默认）
- `dark`: 深色主题

#### 3.1.3 平台兼容性样式
针对iOS、Android和鸿蒙平台的特殊样式处理：
- 安全区适配 (`safe-area-inset-*`)
- 滚动惯性 (`-webkit-overflow-scrolling`)
- 输入框优化 (16px防缩放)
- 毛玻璃效果降级 (`@supports`)

### 3.2 组件库

#### 3.2.1 UI基础组件

| 组件 | 功能 | 特性 |
|------|------|------|
| Button | 按钮 | 支持5种变体、3种尺寸、加载状态 |
| Input | 输入框 | 支持图标、错误状态、成功状态 |
| Modal | 弹窗 | 支持ESC关闭、点击遮罩关闭、动画效果 |
| Card | 卡片 | 支持标题、不同内边距、悬停效果 |
| Toast | 提示 | 支持4种类型、自动关闭、手动关闭 |
| Tabs | 标签页 | 支持3种变体（line/card/pills） |

#### 3.2.2 组件设计原则
- **可组合性**: 组件应设计为可组合的小单元
- **可扩展性**: 通过props支持自定义样式和行为
- **一致性**: 遵循统一的设计令牌和交互模式
- **可访问性**: 支持键盘导航和屏幕阅读器

### 3.3 状态管理

#### 3.3.1 Auth Store (认证状态)
管理用户登录状态和认证信息：
- 支持多种登录方式（账号密码、微信、QQ、Apple、手机号）
- 使用Zustand persist中间件持久化登录状态
- 提供统一的登录/注册/登出方法

#### 3.3.2 Store设计原则
- **单一职责**: 每个store只管理一类状态
- **响应式**: 使用Zustand的响应式更新机制
- **可测试**: 支持mock和测试
- **可扩展**: 支持中间件和组合

### 3.4 路由设计

#### 3.4.1 路由结构

```
/                    # 登录页面
/student             # 学生端首页
/student/chat        # 学生端聊天
/student/relax       # 学生端放松
/student/profile     # 学生端个人中心
/teacher             # 教师端首页
/teacher/chat        # 教师端聊天
/teacher/relax       # 教师端放松
/teacher/profile     # 教师端个人中心
```

#### 3.4.2 路由保护
使用`ProtectedRoute`组件保护需要登录的页面：
- 检查登录状态
- 验证用户角色
- 重定向到对应角色的首页

## 4. 跨平台适配策略

### 4.1 响应式布局
- 使用Tailwind CSS的断点系统实现响应式设计
- 适配360px到1536px的屏幕宽度
- 移动端优先设计

### 4.2 平台特定适配

| 平台 | 适配策略 |
|------|----------|
| iOS | 安全区适配、触摸目标44px、滚动惯性、输入框防缩放 |
| Android | 安全区适配、触摸目标48px、深色模式支持、防点击穿透 |
| 鸿蒙 | 安全区适配、触摸目标44px、能力检测降级策略 |

### 4.3 性能优化
- 代码分割：按路由和模块分割代码
- 懒加载：组件级懒加载
- 资源优化：图片压缩、字体优化
- 缓存策略：HTTP缓存、Service Worker

## 5. 安全考虑

### 5.1 认证安全
- JWT令牌存储在localStorage
- 请求时自动携带Authorization header
- 令牌过期自动刷新或登出

### 5.2 数据安全
- HTTPS加密传输
- 敏感数据加密存储
- 输入验证和XSS防护

### 5.3 代码安全
- TypeScript类型安全
- ESLint代码检查
- 依赖安全扫描

## 6. 开发流程

### 6.1 开发环境
- Node.js 18+
- npm 9+
- Visual Studio Code

### 6.2 命令脚本

| 命令 | 用途 |
|------|------|
| `npm run dev` | 启动开发服务器 |
| `npm run build` | 构建生产版本 |
| `npm run lint` | 代码检查 |
| `npm run preview` | 预览构建结果 |

### 6.3 代码规范
- 使用TypeScript进行类型检查
- 遵循ESLint规则
- 使用Prettier格式化代码
- 组件命名使用PascalCase
- 文件命名使用kebab-case

## 7. 部署方案

### 7.1 构建输出
- 构建产物输出到`dist`目录
- 静态资源使用hash命名，支持缓存
- CSS和JS文件独立打包

### 7.2 部署环境
- 开发环境：本地开发服务器
- 测试环境：测试服务器
- 生产环境：CDN + 静态服务器

### 7.3 跨平台部署
- **Web**: 直接部署静态文件
- **Android**: 使用Capacitor打包为APK
- **iOS**: 使用Capacitor打包为IPA
- **鸿蒙**: 使用WebView容器打包为HAP
