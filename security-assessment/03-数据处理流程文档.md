## 1. 数据处理原则

星屿 StarIsle 涉及未成年人心理健康敏感数据，数据处理严格遵循以下原则，并参照《个人信息保护法》《未成年人保护法》《数据安全法》及 GDPR 等合规要求。

| 原则 | 说明 | 落地措施 |
|------|------|----------|
| 数据最小化 | 仅收集实现业务目的所必需的最少数据 | 注册字段精简、对话内容仅存加密摘要与原文必要部分 |
| 知情同意 | 处理个人信息前明示目的、范围、方式并获得用户/监护人同意 | 注册时展示隐私协议、家长绑定需学生授权 |
| 目的限制 | 数据仅用于声明的目的，不超范围使用 | 学生对话原文不向家长开放，仅用于 AI 分析 |
| 安全保护 | 采取技术与管理措施保障数据机密性、完整性、可用性 | TLS + AES-256-GCM + RBAC + 审计 |
| 未成年人特别保护 | 对未满 14 周岁儿童数据予以特别保护 | 默认最高隐私等级、家长可见性由学生控制 |
| 透明性 | 用户可查询、更正、删除、导出其个人数据 | 提供数据主体权利接口 |
| 问责制 | 明确数据控制者与处理者职责 | 数据安全事件响应流程与责任人 |

## 2. 数据分类

### 2.1 数据分类总表

| 类别 | 数据示例 | 敏感等级 | 存储位置 | 加密方式 |
|------|----------|----------|----------|----------|
| 用户身份信息 | 用户名、密码哈希、角色、学生 / 家长 / 教师标识 | 高 | PostgreSQL `users`、`parent_users` | 密码 BCrypt；其余 AES-256-GCM |
| 情绪数据 | 心情打卡记录、情绪评分、情绪标签 | 高 | PostgreSQL `mood_records` | AES-256-GCM |
| 对话数据 | 学生 AI 对话消息、家长大星对话消息 | 极高 | PostgreSQL `chat_messages`、`parent_chat_messages` | AES-256-GCM |
| 风险评估数据 | 风险等级、风险标签、应急预案记录、急救资源 | 极高 | PostgreSQL `emergency_alerts`、`emergency_resources` | AES-256-GCM |
| 绑定关系数据 | 家长-学生绑定关系、授权状态 | 高 | PostgreSQL `parent_student_bindings` | 字段级加密 |
| 系统日志 | 访问日志、操作日志、异常日志 | 中 | MongoDB / 文件 | 不含敏感字段时明文 |
| 评估数据 | 心理量表评估结果 | 高 | PostgreSQL `assessment_results` | AES-256-GCM |
| 缓存数据 | 会话、Token 黑名单、限流计数 | 中 | Redis | 内存，TTL 自动过期 |

### 2.2 数据敏感性分级

| 等级 | 定义 | 处理要求 |
|------|------|----------|
| 极高 | 直接关联未成年人心理健康与危机状态 | 加密存储 + 严格授权 + 审计 + 最短保留 |
| 高 | 可识别个人身份或情绪状态 | 加密存储 + 授权访问 |
| 中 | 业务辅助数据 | 访问控制 + 脱敏展示 |
| 低 | 公开或匿名化数据 | 基本访问控制 |

## 3. 数据收集

### 3.1 收集方式

| 收集场景 | 收集方式 | 触发端 | 数据流向 |
|----------|----------|--------|----------|
| 用户注册 | 表单主动提交 | 学生 / 教师 / 家长端 | 客户端 → 后端 → PostgreSQL |
| 用户登录 | 表单主动提交 | 全端 | 客户端 → 后端 |
| 心情打卡 | 主动选择情绪与评分 | 学生端 | 客户端 → 后端 → AI 引擎分析 → PostgreSQL |
| AI 对话 | 文本 / 语音输入 | 学生 / 教师 / 家长端 | 客户端 → WebSocket → 后端 → AI 引擎 → PostgreSQL |
| 风险检测 | 由对话 / 打卡内容自动触发 | 后端自动 | AI 引擎 → 后端 → PostgreSQL + 推送家长端 |
| 心理评估 | 量表主动作答 | 学生端 | 客户端 → 后端 → AI 引擎 → PostgreSQL |
| 设备 / 日志 | 自动采集 | 全端 | 客户端 / 网关 → 日志服务 |

### 3.2 收集字段清单

#### 3.2.1 用户身份信息

| 字段 | 类型 | 是否必填 | 用途 | 加密 |
|------|------|----------|------|------|
| `id` | UUID | 系统生成 | 主键 | 否 |
| `username` | 字符串 | 是 | 登录标识 | 否 |
| `password` | 字符串 | 是 | 鉴权 | BCrypt 哈希 |
| `role` | 枚举 | 是 | 角色权限 | 否 |
| `created_at` | 时间戳 | 系统生成 | 审计 | 否 |

#### 3.2.2 情绪数据（mood_records）

| 字段 | 类型 | 用途 | 加密 |
|------|------|------|------|
| `user_id` | UUID | 关联学生 | 否 |
| `mood_score` | 整数 | 情绪评分 1-5 | 否 |
| `mood_tag` | 字符串 | 情绪标签 | AES-256-GCM |
| `note` | 文本 | 备注（可选） | AES-256-GCM |
| `ai_summary` | 文本 | AI 情绪摘要 | AES-256-GCM |
| `created_at` | 时间戳 | 记录时间 | 否 |

#### 3.2.3 对话数据（chat_messages / parent_chat_messages）

| 字段 | 类型 | 用途 | 加密 |
|------|------|------|------|
| `id` | UUID | 主键 | 否 |
| `user_id` | UUID | 用户标识 | 否 |
| `role` | 枚举 | user / assistant | 否 |
| `content` | 文本 | 对话原文 | AES-256-GCM |
| `emotion_analysis` | JSON | 情绪分析结果 | AES-256-GCM |
| `risk_level` | 枚举 | 风险等级 | 否 |
| `created_at` | 时间戳 | 消息时间 | 否 |

#### 3.2.4 风险评估数据（emergency_alerts）

| 字段 | 类型 | 用途 | 加密 |
|------|------|------|------|
| `student_id` | UUID | 关联学生 | 否 |
| `risk_level` | 枚举 | 绿 / 黄 / 橙 / 红 | 否 |
| `trigger_context` | 文本 | 触发上下文摘要 | AES-256-GCM |
| `action_taken` | 文本 | 已采取措施 | AES-256-GCM |
| `parent_notified` | 布尔 | 是否通知家长 | 否 |
| `created_at` | 时间戳 | 触发时间 | 否 |

### 3.3 同意机制

| 同意场景 | 实现方式 | 记录 |
|----------|----------|------|
| 注册 | 用户勾选并同意《隐私政策》《用户协议》 | 数据库记录 `agreed_at` |
| 家长绑定 | 学生在 App 内主动授权 | `parent_student_bindings.authorized = true` |
| 风险推送 | 学生授权时同意《危机干预授权》 | 绑定关系记录授权范围 |
| 数据导出 | 用户主动申请，二次身份验证 | 审计日志 |
| 数据删除 | 用户主动申请，二次身份验证 | 审计日志 |

## 4. 数据存储

### 4.1 存储位置

| 数据库 | 存储内容 | 部署 |
|--------|----------|------|
| PostgreSQL | 用户、绑定、对话、心情、评估、风险、急救资源等核心业务数据 | K8s StatefulSet |
| MongoDB | 系统日志、操作审计、非结构化数据 | K8s StatefulSet |
| Redis | 会话缓存、限流计数、Token 黑名单、热点数据 | K8s StatefulSet |

### 4.2 加密方式

详见《02-安全架构说明文档》第 5 章。核心要点：

| 层级 | 加密方式 |
|------|----------|
| 传输 | TLS 1.2+ |
| 字段 | AES-256-GCM（IV 12 字节、Tag 128 位） |
| 密码 | BCrypt 12 轮 |
| 密钥 | 环境变量 `ENCRYPTION_KEY`（32 字节） |
| 用户级隔离 | SHA-256(userId) 派生密钥 |
| 备份 | 备份文件加密存储 |

### 4.3 保留策略

| 数据类别 | 保留期限 | 到期处理 |
|----------|----------|----------|
| 用户身份信息 | 账号存续期间 + 注销后 30 天 | 自动删除 |
| 对话数据 | 90 天 | 自动归档后删除原文，仅保留摘要 |
| 情绪数据 | 1 年 | 自动聚合为趋势后删除明细 |
| 风险评估数据 | 3 年（合规要求） | 到期自动删除 |
| 心理评估数据 | 2 年 | 到期自动删除 |
| 系统日志 | 6 个月 | 自动清理 |
| 审计日志 | 3 年 | 到期自动归档 |
| 缓存数据 | TTL 自动过期 | Redis 自动清理 |
| 备份数据 | 30 天滚动 | 自动覆盖 |

### 4.4 数据隔离

| 隔离维度 | 实现方式 |
|----------|----------|
| 租户隔离 | 单租户部署，无跨租户数据 |
| 用户隔离 | 用户级密钥派生（SHA-256(userId)），数据查询强制带 userId 条件 |
| 角色隔离 | 学生 / 教师 / 家长数据分表存储（如 `chat_messages` vs `parent_chat_messages`） |
| 隐私隔离 | 学生对话原文对家长绝对不可见，仅情绪摘要可见 |
| 网络隔离 | 数据库仅集群内访问，不暴露公网 |

## 5. 数据传输

### 5.1 传输协议

| 通道 | 协议 | 加密 |
|------|------|------|
| 客户端 ↔ Nginx | HTTPS / WSS | TLS 1.2+ |
| Nginx ↔ 后端 | HTTP | 内网隔离 |
| 后端 ↔ AI 引擎 | HTTP | 内网隔离 |
| 后端 ↔ 数据库 | TCP | 内网隔离，PostgreSQL 可选 SSL |
| 后端 ↔ Redis | TCP | 内网隔离，AUTH 密码 |

### 5.2 加密方式

| 层级 | 算法 |
|------|------|
| 传输加密 | TLS 1.2 / 1.3，使用 ECDHE + AES-GCM |
| 应用层加密 | 业务敏感字段在传输前 / 入库前 AES-256-GCM 加密 |
| 签名 | JWT HMAC-SHA |

### 5.3 端到端加密流程

```
学生端                    后端服务                  AI 引擎
  │                          │                         │
  │ 1.明文消息(本地)         │                         │
  │ ───────TLS WSS────────► │                         │
  │                          │ 2.转发至 AI 引擎         │
  │                          │ ───────HTTP───────────► │
  │                          │                         │ 3.情绪分析+风险检测
  │                          │                         │ 4.生成 AI 回复
  │                          │ 5.返回分析结果+回复       │
  │                          │ ◄────────────────────── │
  │                          │ 6.AES-256-GCM 加密      │
  │                          │ 7.写入 chat_messages    │
  │                          │ 8.生成情绪摘要(脱敏)     │
  │ 9.返回 AI 回复            │                         │
  │ ◄──────────────────────  │                         │
  │                          │                         │
  │                          │ 10.家长查询时仅返回摘要  │
  │                          │    不返回对话原文         │
```

> 关键点：家长端访问后端时，后端仅返回学生的情绪趋势摘要，**对话原文永不返回**给家长端。

## 6. 数据使用

### 6.1 使用场景

| 场景 | 使用数据 | 使用方 | 目的 |
|------|----------|--------|------|
| 用户鉴权 | 用户名、密码哈希、角色 | 后端 | 登录与权限控制 |
| AI 对话 | 对话消息 | AI 引擎 | 生成回复、情绪分析 |
| 风险检测 | 对话内容、情绪数据 | AI 引擎 + 后端 | 识别自杀 / 自残风险 |
| 心情打卡分析 | 情绪数据 | AI 引擎 | 生成情绪摘要与趋势 |
| 家长查看 | 情绪摘要、风险等级 | 家长端 | 关注孩子心理状态 |
| 教师查看 | 所辖学生风险概览 | 教师端 | 班级心理状况监测 |
| 应急预案 | 风险等级、急救资源 | 后端 + 家长端 | 触发危机干预 |
| 量表评估 | 评估作答 | AI 引擎 | 心理健康评估 |
| 系统运维 | 日志、监控 | 运维 | 故障排查与性能优化 |

### 6.2 访问控制

| 控制点 | 机制 |
|--------|------|
| 接口访问 | JWT 鉴权 + RBAC 端点授权 |
| 数据访问 | 服务层强制注入 userId / 角色条件 |
| 家长可见性 | 学生授权开关控制，`parent_student_bindings.authorized` |
| 教师可见性 | 仅查看所辖学生，且仅风险概览，不查看对话原文 |
| 运维访问 | 跳板机 + 审计，不直接访问生产数据 |
| 数据导出 | 二次身份验证 + 审计日志 |

### 6.3 AI 数据处理

| 处理环节 | 说明 | 隐私保护 |
|----------|------|----------|
| 对话理解 | AI 引擎解析用户输入 | 仅在内存中处理，不持久化中间结果 |
| 情绪分析 | 基于对话与打卡内容 | 仅持久化摘要，不持久化原始分析过程 |
| 风险检测 | 关键词 + 语义模型 | 风险标签持久化，触发上下文加密存储 |
| 回复生成 | 基于学生 / 家长独立 System Prompt | 学生与家长对话完全隔离 |
| 模型训练 | 当前不基于用户数据训练 | 数据不流出生产环境 |

## 7. 数据共享

### 7.1 三端数据流关系

```
┌─────────────┐                ┌─────────────┐                ┌─────────────┐

└──────┬──────┘                └──────┬──────┘                └──────┬──────┘

       │ ① 心情打卡 + AI 对话          │                              │
       │ ──────────────────────────► │                              │
       │                              │ ② 经学生授权判断               │
       │                              │ ③ 仅推送情绪摘要 + 风险等级    │
       │                              │ ──────────────────────────► │

       │ ──────────────────────────► │ ──► 实时控制家长可见性          │
       │                              │                              │
       │                              │ ⑤ 橙 / 红风险自动推送          │
       │                              │    触发应急预案                │
       │                              │ ──────────────────────────► │
       │                              │                              │
       │                              │ ⑥ 家长与大星对话（独立Prompt）  │
       │                              │ ◄────────────────────────── │
       │                              │ ⑦ 返回大星回复                │
       │                              │ ──────────────────────────► │
       │                              │                              │
       │   ✗ 对话原文对家长绝对不开放   │                              │
       │   ✓ 家长仅见情绪趋势摘要      │
```

### 7.2 共享规则

| 共享场景 | 共享数据 | 共享条件 | 共享对象 |
|----------|----------|----------|----------|
| 学生 → 家长：情绪摘要 | 情绪趋势、心情打卡摘要 | 学生已授权绑定 | 已绑定的家长 |
| 学生 → 家长：风险预警 | 风险等级（橙 / 红）、应急预案 | 自动触发，学生授权范围内 | 已绑定的家长 |
| 学生 → 教师：风险概览 | 班级学生风险等级统计 | 教师为该班级负责教师 | 教师 |
| 家长 → AI：大星对话 | 家长输入消息 | 家长已登录 | AI 引擎（独立 Prompt） |
| 学生 → AI：小星对话 | 学生输入消息 | 学生已登录 | AI 引擎（独立 Prompt） |
| 系统内部：风险检测 | 对话内容、情绪数据 | 自动触发 | AI 引擎 → 后端 |
| 第三方共享 | 无 | 不向第三方共享用户数据 | — |

### 7.3 隐私隔离规则

| 隔离项 | 规则 |
|--------|------|
| 学生对话原文 | 对家长、教师绝对不可见 |
| 学生情绪摘要 | 仅在学生授权时对家长可见 |
| 家长对话内容 | 对学生、教师绝对不可见 |
| 教师可见范围 | 仅所辖学生的风险等级统计，不可见对话与详细情绪 |
| 风险推送范围 | 仅推送至已绑定且授权的家长 |
| 授权控制权 | 完全由学生掌握，可随时开启 / 关闭 |
| AI 引擎隔离 | 学生对话与家长对话使用独立 System Prompt 与独立存储表 |

## 8. 数据销毁

### 8.1 销毁场景与流程

| 场景 | 触发方 | 流程 | 完成时限 |
|------|--------|------|----------|
| 用户主动注销 | 用户 | 申请 → 二次身份验证 → 软删除（30 天保留期）→ 硬删除 | 30 天后自动硬删除 |
| 用户主动删除对话 | 用户 | 申请 → 删除指定对话 → 审计日志 | 立即 |
| 过期自动删除 | 系统定时任务 | 按保留策略扫描 → 删除过期数据 → 审计日志 | 每日执行 |
| 日志清理 | 系统定时任务 | 按保留期清理日志 → 归档 | 每日执行 |
| 缓存过期 | Redis TTL | 自动过期 | 实时 |
| 备份覆盖 | 备份系统 | 30 天滚动覆盖 | 每次备份 |
| 应急销毁 | 安全管理员 | 评估 → 审批 → 物理销毁 → 审计 | 按事件流程 |

### 8.3 销毁验证

| 验证项 | 方式 |
|--------|------|
| 数据库记录 | 查询确认记录已删除 |

| 缓存 | 确认 Redis 对应 key 已过期 / 删除 |
| 日志 | 审计日志记录销毁操作 |
| AI 引擎 | 确认无中间结果残留 |

## 10. 数据安全事件响应

### 10.1 事件分类

| 等级 | 定义 | 示例 | 响应时限 |
|------|------|------|----------|
| 特别重大 (I 级) | 大规模用户敏感数据泄露 | 对话数据库被拖库、加密密钥泄露 | 立即 |
| 重大 (II 级) | 部分用户敏感数据泄露 | 单用户数据越权访问、AI 引擎被注入 | 1 小时内 |
| 较大 (III 级) | 系统安全事件未造成数据泄露 | 暴力破解未遂、DDoS 攻击 | 4 小时内 |
| 一般 (IV 级) | 一般安全告警 | 异常登录、限流触发 | 24 小时内 |
